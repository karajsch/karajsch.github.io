Category Pages

Refocusing the Homepage Around User Intent

Simplifying a marketing-heavy surface into a functional entry point

UX Strategy • Front-End Architecture • Design Systems • Performance Optimization

Simplified a high-maintenance, low-engagement homepage into a focused, scalable entry point aligned with user intent.

Most users arrived with specific goals and moved quickly into product and category pages - making speed and clarity more valuable than promotional content.

I led a restructuring effort to simplify the experience, align content with user behavior, and support scalable content management.

Homepage Before

The homepage had evolved into a dense, marketing-driven surface with competing priorities, low engagement elements, and high maintenance overhead.

Homepage After

The redesign reduced visual density and prioritized direct pathways into product discovery.

Context

The homepage had evolved into a dense, marketing-driven surface with competing priorities and low engagement.

Users primarily used it as a pass-through, while internally it functioned as a marketing surface with frequent updates and inconsistent structure.

  • Multiple stakeholders contributing content
  • Legacy templates and inconsistent structure
  • Manual updates requiring developer support

Strategy

The strategy focused on simplifying the homepage to reflect real user behavior - prioritizing product discovery, reducing noise, and introducing scalable content patterns.

  • Prioritize product discovery
  • Remove low-performing content
  • Introduce repeatable structures

Content Simplification

Low-engagement elements were removed and high-value pathways were elevated, reducing cognitive load and clarifying the page structure.

  • Removed low-engagement marketing content (value props, mission, featured brands)
  • Eliminated carousel banners in favor of focused content
  • Removed redundant navigation and secondary category structures
  • Prioritized direct access to product categories

System & Template Evolution

I partnered with developers to restructure the homepage into modular, reusable content patterns.

  • Introduction of templated banner components
  • Standardized layouts for Knowledge Center and webinar content
  • Modular featured product and category sections
  • Improved integration between content types

This reduced reliance on one-off implementations and improved consistency across updates.

Experimentation

Targeted A/B tests on homepage banners informed decisions around motion, layout, and content prioritization, reinforcing the shift toward simpler, more focused content.

Outcomes

  • Simplified pathways into product discovery
  • Reduced visual and cognitive load
  • Removed legacy code and improved performance
  • Enabled scalable, low-maintenance content updates

Building Scalable Front-End Components for Event Content

Unifying fragmented experiences into a structured, maintainable system

Design Systems • Front-End Architecture • UX Strategy • Cross-functional Collaboration

Transformed fragmented event content into a unified, templated system that improved discoverability and eliminated manual page creation.

Event content across the site was fragmented across multiple page types, with inconsistent structure and limited visibility. I led an effort to unify these experiences and introduce a scalable system for managing event content.

Events Before

Event pages had spread out over the website with different entry points for different events (webinars, in-person events, summits, trade shows), all created manually in HTML.

Events After

The overhaul produced a clean, streamlined single events page with a table listing all events, with the different types of events now data-types that can be sorted and filtered according to event status.

Context

Events existed across several disconnected areas - including webinars, trade shows, and in-person events - each with different structures and entry points.

The primary events page surfaced only a small subset of content, while other event types were buried within separate sections like the Knowledge Center or standalone pages.

  • No unified view of all event types
  • Inconsistent page structures across formats
  • Limited discoverability and navigation between events

Problem

Event pages were built as large blocks of static HTML, requiring manual duplication and developer involvement for updates. This created inefficiencies in both content management and long-term maintenance.

Strategy

The approach focused on unifying event content and introducing structured, reusable components that could support multiple event types while simplifying content creation.

System Design

I designed a flexible component system to support different event formats, along with a centralized index page to unify discovery and navigation.

  • Defined five event types: webinar, event (basic & deluxe), and tradeshow (basic & deluxe)
  • Designed reusable templates with structured inputs for each content type
  • Created a centralized events index displaying all event types in a sortable format

I created front-end prototypes for each component and partnered with developers to implement the underlying system and data structure.

Outcomes

  • Unified previously fragmented event experiences into a single system
  • Improved discoverability across all event types
  • Enabled non-technical users to create and manage event pages
  • Reduced reliance on manual HTML updates and developer support
  • Established a scalable foundation for future event content
